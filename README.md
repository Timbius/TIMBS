# Название проекта: Сайт Барбершопа

## 1. Тема и концепция

**Barber Factory** — это современное веб-приложение для онлайн-записи в барбершоп. Платформа позволяет клиентам выбрать услугу, мастера и удобное время посещения без необходимости звонка администратору. Пользователи могут управлять своими записями и личными данными через персональный кабинет, просматривать каталог услуг, их описание и стоимость, самостоятельно выбрать мастера и удобное время для записи. Приложение упрощает процесс бронирования и записи, делая взаимодействие с барбершопом быстрым и удобным.

**Аудитория:**
Приложение создано для мужчин любых возрастов, которые знают толк в уходе за собой и регулярно посещают барбершоп; для клиентов, которые предпочитают онлайн-запись вместо телефонных звонков или постоянных очередей; для людей, которые ценят своё время и комфорт.

### Ключевой функционал:

* **Аутентификация и регистрация пользователей**
    * Создание аккаунта (email/телефон + пароль).
    * Авторизация в системе.
    * Восстановление пароля.
* **Выбор услуг и мастеров**
    * Просмотр списка доступных услуг (стрижка, бритьё, укладка и др.).
    * Отображение стоимости и продолжительности услуги.
    * Выбор конкретного мастера с просмотром его профиля и рейтинга.
* **Онлайн-запись на услугу**
    * Выбор удобной даты и времени из доступных слотов.
    * Подтверждение записи.
    * Просмотр и отмена предстоящих записей.
* **Личный кабинет пользователя**
    * Редактирование персональных данных.
    * Просмотр истории посещений.
    * Управление активными записями.

## 2. Структура приложения

1.  **Главная** (`/`) — основная страница сайта.
2.  **Каталог услуг** (`/catalog`) — список услуг.
3.  **Подробная страница услуги** (`/catalog/:id`) — подробное описание услуги и цена.
4.  **Страница мастеров** (`/barbers`) — список мастеров.
5.  **Подробная страница мастера** (`/barbers/:id`) — подробная информация про мастера.
6.  **Страница онлайн-записи** (`/records`) — страница для записи на услугу.
7.  **Поиск** (`/search`) — поиск по услуге или по мастеру.
8.  **Аутентификация** (`/auth/*`) — регистрация / вход / восстановление пароля.
9.  **Личный кабинет** (`/profile`) — кабинет с информацией пользователя.
10. **Админ панель** (`/admin`) — раздел доступный только пользователям с ролью администратор, позволяющая управлять учетными записями.

## 3. Технологический стек

### Frontend
* **Vanilla JavaScript** (Без фреймворков).
* **Причины выбора:** Демонстрация базовых навыков разработки, контроль архитектуры.

### Backend
* **Node.js & Express.js**
* **Причины выбора Node.js:** Использование JavaScript и на frontend, и на backend — единая экосистема разработки; высокая производительность при работе с асинхронными запросами (поиск, авторизация, работа с БД); REST API.
* **Причины выбора Express.js:** Лёгкий и минималистичный фреймворк; удобная маршрутизация; поддержка middleware; простота REST API.

### База данных
* **MySQL**
* **Причины выбора:** Реляционная структура данных, поддержка связей, стабильность, производительность.

### Инфраструктура
* **Frontend:** Static hosting.
* **Backend:** Node.js Server.
* **Database:** MySQL.

## 5. Визуальное проектирование интерфейса

В данном разделе представлено текстовое описание интерфейса приложения. Каждая страница описана с указанием типа доступа, структуры блоков и динамического контента.

### 1. Главная страница (`/`)
* **Тип доступа:** Публичный.
* **Основные блоки:**
    * **Header:** Логотип, Навигация (Услуги, Мастера, Запись), Кнопка поиска, Кнопка «Войти» или Аватар.
    * **Hero-блок:** Баннер с акцией/слоганом, Кнопки «Записаться онлайн» и «Смотреть услуги».
    * **Секция «Популярные услуги»:** Карточки с названием, описанием и ценой.
    * **Секция «Наши мастера»:** Превью (фото, имя, специализация).
    * **Footer:** Контакты, адрес, соцсети.
* **Динамический контент:**
    * Популярные услуги: `GET /api/services?popular=true`
    * Топ-мастера: `GET /api/barbers?top=true`

### 2. Каталог услуг (`/catalog`)
* **Тип доступа:** Публичный.
* **Основные блоки:** Фильтры (поиск, категории, сортировка), Сетка услуг (карточки с кнопкой «Подробнее»).
* **Динамический контент:** `GET /api/services` (фильтрация через query-параметры).

### 3. Подробная страница услуги (`/catalog/:id`)
* **Тип доступа:** Публичный.
* **Основные блоки:** Полное описание, длительность, цена, кнопка «Записаться», блок «Рекомендуемые мастера».
* **Динамический контент:** `GET /api/services/:id`, `GET /api/barbers?service=:id`

### 4. Страница мастеров (`/barbers`)
* **Тип доступа:** Публичный.
* **Основные блоки:** Фильтры (специализация, рейтинг, опыт), Список мастеров.
* **Динамический контент:** `GET /api/barbers`

### 5. Подробная страница мастера (`/barbers/:id`)
* **Тип доступа:** Публичный.
* **Основные блоки:** Профиль (фото, опыт, описание), Список услуг мастера, Отзывы клиентов.
* **Динамический контент:** `GET /api/barbers/:id`, `GET /api/barbers/:id/services`, `GET /api/barbers/:id/reviews`

### 6. Страница онлайн-записи (`/records`)
* **Тип доступа:** Авторизованный пользователь.
* **Основные блоки:** Форма записи (выбор услуги, мастера, даты, времени, комментарий), Информация о записи (цена, длительность).
* **Динамический контент:** `GET /api/services`, `GET /api/barbers`, `GET /api/schedule?barber=:id&date=:date`, `POST /api/records`

### 7. Поиск (`/search`)
* **Тип доступа:** Публичный.
* **Основные блоки:** Строка поиска, Результаты поиска (вкладки «Услуги» и «Мастера»).
* **Динамический контент:** `GET /api/search?q=keyword`

### 8. Аутентификация (`/auth/*`)
* **Тип доступа:** Публичный.
* **Формы:** Вход, Регистрация, Восстановление пароля.
* **Динамический контент:** `POST /api/auth/login`, `POST /api/auth/register`, `POST /api/auth/forgot-password`.

### 9. Личный кабинет (`/profile`)
* **Тип доступа:** Авторизованный пользователь.
* **Блоки:** Информация о пользователе, Мои записи (статусы, отмена), История посещений.
* **Динамический контент:** `GET /api/users/me`, `GET /api/users/me/records`, `PUT /api/users/me`, `DELETE /api/records/:id`

### 10. Админ панель (`/admin`)
* **Тип доступа:** Администратор.
* **Блоки:** Меню управления (услуги, мастера, записи, пользователи), Таблицы данных с кнопками редактирования/удаления.
* **Динамический контент:** CRUD запросы к `/api/services`, `/api/barbers`, `/api/users`.

## 6. Схема данных и API

### Основные модели данных

#### 1. Модель пользователя (User)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| id | integer | Уникальный идентификатор |
| name | string | Имя пользователя |
| email | string | Электронная почта (уникальная) |
| password | string | Хэш пароля |
| role | string | Роль (user / admin) |
| createdAt | datetime | Дата регистрации |

#### 2. Основная сущность — Услуга
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| id | integer | Уникальный идентификатор |
| title | string | Название услуги |
| description | text | Описание услуги |
| price | decimal | Цена |
| imageUrl | string | Ссылка на изображение |
| category | string | Категория |
| createdAt | datetime | Дата добавления |

### Связи между моделями
* Один пользователь может создать несколько записей на различные услуги.
* Связь реализуется через таблицу **records** (Many-to-Many):

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| **id** | integer | Идентификатор записи |
| **userId** | integer | ID пользователя |
| **serviceId** | integer | ID услуги |
| **barberId** | integer | ID мастера |
| **date** | datetime | Дата и время посещения |

## 7. API

### Аутентификация

#### Регистрация
`POST /api/auth/register`

**Тело запроса:**
```
{
  "name": "Иван",
  "email": "ivan@mail.com",
  "password": "123456"
}
```
**Ответ:**

```
{
  "token": "jwt_token",
  "user": {
    "id": 1,
    "name": "Иван",
    "email": "ivan@mail.com",
    "role": "user"
  }
}
```

### Вход
`POST /api/auth/login`

**Тело запроса:**
```
{
  "email": "ivan@mail.com",
  "password": "123456"
}
```

**Ответ:**
```
{
  "token": "jwt_token",
  "user": {
    "id": 1,
    "name": "Иван",
    "role": "user"
  }
}
```
### Услуги (Services)
#### Получение списка услуг
`GET /api/services`

Query параметры:

category — фильтр по категории

sort — сортировка (price_asc, price_desc)

popular=true — только популярные

**Ответ:** 

```
  {
    "id": 1,
    "title": "Стрижка классическая",
    "price": 1500,
    "imageUrl": "/images/service1.jpg",
    "isPopular": true
  }
  ```

### Получение детальной информации об услуге
`GET /api/services/:id`

**Пример:** `/api/services/1`

**Ответ:**
```
{
  "id": 1,
  "title": "Стрижка классическая",
  "description": "Полное описание услуги, включая техники и инструменты",
  "price": 1500,
  "imageUrl": "/images/service1.jpg",
  "category": "Стрижки",
  "isPopular": true,
  "createdAt": "2026-03-14T12:00:00Z"
}
```

### Записи (Records)
#### Создание записи
`POST /api/records`

**Тело запроса:**
```
{
  "serviceId": 1,
  "barberId": 2,
  "date": "2026-03-20T14:00:00Z"
}
```
**Ответ:**
```
{
  "message": "Запись успешно создана",
  "recordId": 105
}
```

### Доступы
#### Роль
* User
##### Возможности
* Просмотр каталога
* создание записи на услугу
#### Роль
* Admin
##### Возможности
* Управление услугами и мастерами (CRUD)
* управление пользователями
